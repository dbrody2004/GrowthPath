# Technical Architecture

## Overview

GrowthPath is an npm workspaces monorepo with three runtime apps and two shared packages.

```
apps/
  api/      Express REST API
  ui/       React SPA
  worker/   Background job consumer
packages/
  config/   Zod-based env parsing
  shared/   DTOs, queue names, constants
```

## Data flow

1. The UI calls the API over HTTP (`/api/*`, `/health`).
2. The API persists data in MongoDB and publishes jobs to RabbitMQ.
3. The worker consumes jobs from the main queue and writes results back to MongoDB when needed.
4. Object storage uses MinIO locally and in production via the AWS S3 SDK.

## Queue topology

- Exchange: `growthpath.jobs`
- Main queue: `growthpath.jobs.main`
- Dead-letter exchange: `growthpath.jobs.dlx`
- Dead-letter queue: `growthpath.jobs.dlq`

## Local vs production

| Concern | Local | Production |
| --- | --- | --- |
| Apps | Host processes via `npm run dev:apps` | Docker containers |
| Infra | `docker-compose.local.yml` | `docker-compose.prod.yml` |
| TLS | None (Vite dev server) | Nginx + Certbot in UI container |
| MongoDB | `127.0.0.1:27118` | `mongo:27017` |
| RabbitMQ | `127.0.0.1:5674` | `rabbitmq:5672` |
| MinIO | `127.0.0.1:9004` | `minio:9000` |

## Auth model

The API uses a session-token header (`x-session-token`) stored as a hashed value on the user document. Product-specific auth can extend this without changing the scaffold shape.
