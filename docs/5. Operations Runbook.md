# Operations Runbook

## Health checks

| Component | Check |
| --- | --- |
| API | `GET /health` |
| Worker | Docker process liveness / logs |
| MongoDB | `docker compose ... ps` health status |
| RabbitMQ | Management UI or `rabbitmq-diagnostics ping` |
| MinIO | `/minio/health/live` |
| UI | HTTPS homepage and `/health` proxy |

## Logs

Local infra:

```bash
npm run docker:logs
```

Production:

```bash
docker compose -f docker-compose.prod.yml logs -f api worker ui
```

## Restart services

```bash
docker compose -f docker-compose.prod.yml restart api worker ui
```

## Backup data

Persistent volumes live under:

- `data/mongo`
- `data/rabbitmq`
- `data/minio`
- `data/letsencrypt`

Back up these directories before major upgrades.

## Rotate secrets

1. Generate new secrets with `npm run generate:secrets`.
2. Update `.env`.
3. Redeploy with `npm run deploy:prod`.
4. Invalidate active sessions by clearing `sessionToken` fields if needed.

## Queue recovery

Failed jobs land in `growthpath.jobs.dlq`. Inspect messages in RabbitMQ management UI and requeue after fixing the handler.

## Seed admin password hash

```bash
node scripts/hash-seed-admin-password.mjs 'your-password'
```

Use the output when manually inserting admin users in MongoDB.

## Troubleshooting

| Symptom | Likely cause | Action |
| --- | --- | --- |
| API `degraded` health | MongoDB down | `npm run docker:up`, verify `MONGODB_URI` |
| Jobs not processing | Worker not running or RabbitMQ down | Check worker logs and queue bindings |
| UI 502 on `/api` | API container unhealthy | Inspect api logs and env overrides |
| Certbot failures | DNS or port 80 blocked | Verify domain A record and firewall |
