# Production Deployment

## Prepare environment

1. Copy `.env.example` to `.env` on the target VM.
2. Set production secrets:

```bash
npm run generate:secrets
```

3. Update production values:

- `DEFAULT_DOMAIN`
- `CERTBOT_EMAIL`
- `MONGODB_URI`, `RABBITMQ_URL`, `S3_ENDPOINT` (or rely on URI rewrite)
- `REGISTRY`, `IMAGE_NAMESPACE`, `TAG`

## Build images

```bash
PUSH=true TAG=$(git rev-parse --short HEAD) npm run build:prod
```

Environment variables for `build-prod.sh`:

| Variable | Default | Purpose |
| --- | --- | --- |
| `REGISTRY` | `ghcr.io` | Container registry |
| `IMAGE_NAMESPACE` | `growthpath` | Image namespace |
| `IMAGE_PREFIX` | `growthpath` | Image name prefix |
| `TAG` | git SHA | Image tag |
| `PUSH` | `false` | Push to registry |
| `PUSH_LATEST` | `true` | Also tag `latest` |
| `PLATFORMS` | `linux/amd64,linux/arm64` | buildx platforms |

## Deploy

On the production host:

```bash
npm run deploy:prod
```

Optional swap creation for small VMs:

```bash
CREATE_SWAP=true npm run deploy:prod
```

The deploy script:

1. Requires `.env`
2. Creates `data/` directories
3. Writes `.env.prod.override` with Docker service hostnames
4. Pulls app images
5. Starts `docker-compose.prod.yml`

## TLS

Set `ENABLE_CERTBOT=true` on the UI container environment for live certificate issuance. The UI entrypoint bootstraps a self-signed cert first, then attempts Certbot renewal on a background loop.

## Verify

```bash
curl -k https://your-domain/health

```
