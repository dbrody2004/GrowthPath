# Production Deployment

## Prepare environment

1. Copy `.env.example` to `.env` on the target VM.
2. Set production secrets:

```bash
npm run generate:secrets
```

3. Update production values:

- `DEFAULT_DOMAIN`
- `CERTBOT_EMAIL`
- `MONGODB_URI`, `RABBITMQ_URL`, `S3_ENDPOINT` (or rely on URI rewrite)
- `REGISTRY`, `IMAGE_NAMESPACE`, `TAG`

## Build images

Build and push the app images (api, worker, ui):

```bash
PUSH=true TAG=$(git rev-parse --short HEAD) npm run build:prod
```

Build and push the marketing site image:

```bash
PUSH=true TAG=$(git rev-parse --short HEAD) npm run build:marketing:prod
```

Both scripts live at the repo root and produce multi-arch images
(`linux/amd64` for Linux hosts and `linux/arm64` for Apple Silicon / Mac).

Environment variables for `build-prod.sh` and `build-marketing.sh`:

| Variable | Default | Purpose |
| --- | --- | --- |
| `REGISTRY` | `ghcr.io` | Container registry |
| `IMAGE_NAMESPACE` | your GitHub username or org | GHCR owner (must match an account you can push to) |
| `IMAGE_PREFIX` | `growthpath` | Image name prefix |
| `TAG` | git SHA | Image tag |
| `PUSH` | `false` | Push to registry |
| `PUSH_LATEST` | `true` | Also tag `latest` |
| `PLATFORMS` | `linux/amd64,linux/arm64` | buildx platforms |

## Deploy

On the production host:

```bash
npm run deploy:prod
```

Optional swap creation for small VMs:

```bash
CREATE_SWAP=true npm run deploy:prod
```

The deploy script:

1. Requires `.env`
2. Creates `data/` directories
3. Writes `.env.prod.override` with Docker service hostnames
4. Pulls app images
5. Starts `docker-compose.prod.yml`

## Deploy marketing site

The marketing site deploys independently on its own host (it binds ports
80/443). Deploy always pulls the image from the registry.

**1. Push from your build machine:**

```bash
echo "$GITHUB_TOKEN" | docker login ghcr.io -u YOUR_GITHUB_USERNAME --password-stdin
PUSH=true TAG=$(git rev-parse --short HEAD) ./build-marketing.sh
```

**2. Deploy on the server:**

Add `GITHUB_USERNAME` and `GITHUB_TOKEN` (with `read:packages`) to `.env`,
or log in once with `docker login`, then:

```bash
./deploy-marketing.sh
```

Ensure `TAG` in `.env` matches the tag you pushed (default `latest` if
`PUSH_LATEST=true` on build). Set `IMAGE_NAMESPACE` to your GitHub username
or org (e.g. `dbrody2004`), not the app name — GHCR paths look like
`ghcr.io/OWNER/growthpath-marketing:latest`.

The marketing deploy script:

1. Requires `.env`
2. Creates the `data/letsencrypt` and `data/certbot-www` directories
3. Logs in to GHCR when `GITHUB_USERNAME` and `GITHUB_TOKEN` are set
4. Pulls the marketing image from the registry
5. Starts `docker-compose.marketing.yml`

## TLS

Set `ENABLE_CERTBOT=true` on the UI container environment for live certificate issuance. The UI entrypoint bootstraps a self-signed cert first, then attempts Certbot renewal on a background loop.

## Verify

```bash
curl -k https://your-domain/health
curl -k https://your-domain/api/auth/health
```
